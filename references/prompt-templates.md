# 攻略提示词模板

Use these prompts when this skill needs to guide another Codex run or when the user wants a copy-paste prompt.

## 一键完整提示词

```text
使用 $xhs-travel-planner，帮我做一个【目的地】【天数】旅行计划。

要求：
1. 只使用小红书真实笔记作为来源。
2. 必须第一步打开浏览器到小红书，让我扫码登录；不要索要账号密码、Cookie、验证码。
3. 登录确认后，再搜索并打开真实笔记：1天至少8条，2-4天至少15条，5天及以上至少20条；覆盖路线、美食、餐厅、夜市、避坑、住宿区域、交通。
4. 只从已登录小红书浏览器会话读取来源；不要用非小红书网页替代。
5. 优先选择当前搜索结果中可见点赞/收藏较高、内容具体的笔记；高赞也必须与其他独立笔记交叉确认。
6. 每条来源保留：搜索词、笔记链接、标题、作者、发布时间、可读摘要、可见点赞数/收藏数/评论数；看不到就填 null，不得编造。
7. 提取地点、餐厅、推荐菜、避坑、排队、预约、闭店、价格、交通风险。
8. 把模糊、单一弱来源或图片不可读的信息标成 candidate，不要当作强推荐。
9. 生成每日顺路路线，用高德地图链接校验地点顺序。
10. 最终输出一个可部署、手机可打开的单文件 travel-plan.html。

我的信息：
目的地：
天数/日期：
出行人数：
住宿区域：
旅行风格：
必须去：
不想去/忌口：
预算：
```

## 小红书采集提示词

```text
请先采集小红书真实笔记，不要生成行程。

第一步必须打开浏览器到小红书并让我扫码登录。登录确认前不要搜索、不要生成、不要使用其他网页来源。

搜索词至少包括：
- 【目的地】 行程
- 【目的地】 美食
- 【目的地】 避坑
- 【目的地】 夜市
- 【目的地】 本地人推荐
- 【目的地】 住宿区域

采样要求：
- 1天至少打开8条真实笔记，2-4天至少15条，5天及以上至少20条。
- 每种核心搜索意图都打开多条笔记，不要只采第一屏或单一关键词。
- 优先打开可见点赞/收藏较高且包含明确地点、分店或体验细节的笔记。
- 高赞不能替代核验；重点推荐和避坑信息尽量由不同作者的笔记交叉支持。

输出一个 sources 表格：
id / 搜索词 / 标题 / 作者 / 日期 / 链接 / 点赞 / 收藏 / 评论 / 可读摘要 / 可信度 / 可提取地点 / 避坑信息

如果正文或互动数据读不到，相关字段填 null；信息不充分时把可信度降为 candidate，不得补写不存在的数字或体验。
```

## 信息抽取提示词

```text
基于已采集的小红书 sources，抽取旅行计划数据。

输出：
1. places：地点、餐厅、夜市、住宿区域、景点。
2. warnings：避坑、排队、价格、预约、分店差异、交通问题。
3. route candidates：适合放在同一天的地点组合。

规则：
- 每个 place/warning 必须有 sourceIds。
- 分店不确定时不要合并成 confirmed。
- 多条笔记冲突时保留冲突说明。
- 用户体验差的地点要进入 warnings，不要硬塞进推荐路线。
```

## 行程生成提示词

```text
根据 places 和 warnings 生成【天数】每日路线。

路线规则：
- 每天 3-5 个主节点即可，手机页面要好扫读。
- 优先顺路，少折返；同片区集中安排。
- 每天至少包含一个可替换备选。
- 饭点不要连续安排高排队餐厅。
- 高风险地点放进避坑区或候选项。
- 每个 item 必须带 sourceIds、placeId、timeBlock、notes、warningIds。
```

## HTML 生成提示词

```text
请把 itinerary JSON 生成一个单文件 travel-plan.html。

页面要求：
- 手机 H5 风格，不做普通网页落地页，也不要做假手机状态栏、返回、收藏、圆形分享按钮。
- 页面模板使用 Tailwind CDN，布局遵循 skill 自带 `scripts/create_static_html.py` 的 H5 模板：`max-w-md` 居中白色页面、深色封面、蓝色 tabs、精简时间轴、底部预算栏。
- 顶部是大图 hero：旅行计划标题、日期、人数、风格 chips、`保存页面`按钮。
- 点击`保存页面`时，用浏览器下载当前单文件 HTML，文件名为 travel-plan.html；不能用 alert 代替。
- 行程区域是 hero 下方的白色内容区。
- 每日路线用 Day Tab 切换，当前日期有蓝色短下划线。
- Day Tab 下方显示今日路线概览：地点 A → 地点 B → 地点 C。
- 路线用时间轴展示：左侧时间和圆点，右侧地点卡片，卡片含缩略图、类别标签、区域/时长/费用。
- 地点卡片可展开，显示推荐理由、提示、来源、避坑。
- 高德地图按钮为主，百度地图为备选。
- 每个地点的展开详情内显示对应的小红书来源跳转；不要在页面末尾单独列出全部来源。
- 不要生成“数据边界”区块；必要的不确定提示只放在对应地点或避坑提醒内。
- 底部固定操作栏：预算总计、查看预算明细；地图入口放在展开后的地点详情中。
- 不需要本地服务，不要输出 Vite dev server 作为最终成果。
- 文件名必须是 travel-plan.html；部署到 GitHub Pages 时改名为 index.html。
```

## GitHub Pages 发布提示词

```text
请将 travel-plan.html 作为 GitHub Pages 静态页面发布。

发布流程：
1. 把 travel-plan.html 复制为 index.html。
2. 上传到目标 public 仓库根目录，或 docs/ 目录。
3. 确认 GitHub Pages 使用该分支和目录。
4. 返回最终手机可打开链接。

不要发布本地 dev server 地址。
```
